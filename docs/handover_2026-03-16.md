## 4. Critical Context

### Architectural decisions:
- **Moti for animations** — All meditation animations use `moti` (React Native animation library). It wraps Reanimated 2 with a declarative API. Works well for opacity/scale/translate but has limitations (e.g., `AnimatePresence exitBeforeEnter` can be flaky)
- **Absolute positioning for layout stability** — The circle zone (`position: 'absolute', top: 0, height: 50%`) and text zone (`top: 50%, height: 22%`) are completely decoupled so text changes never shift the circle
- **State machine pattern** — Session flow is a pure reducer in `TadabburSessionEngine.ts`. Each phase transition is an action dispatch. This is clean and testable
- **Audio via react-native-track-player** — Verse recitation uses RNTP (same as the main app). The `playVerseRecitation()` function returns a Promise that resolves when audio completes (via event + polling fallback)

### Tricky bugs encountered:
1. **Timer inflation** — `getSessionDuration()` was using live `Date.now()` every render. On the closing screen, minutes would keep climbing. Fixed by freezing at `closingTime`
2. **Circle jumping** — The `size` prop toggling 160→170 caused a layout recalculation. Even though the circle zone is absolutely positioned, the `scale` animation base changed, causing a visual jump
3. **Audio completion detection** — RNTP's `PlaybackQueueEnded` event is unreliable on some iOS versions. Added a polling fallback (checks player state every 2s)
4. **Blob shapes** — Asymmetric `borderTopLeftRadius` vs `borderTopRightRadius` values (e.g., `0.6 * SCREEN_WIDTH` vs `0.8 * SCREEN_WIDTH`) create irregular blob shapes instead of smooth waves

### User preferences expressed:
- Wants a **Headspace/Calm-quality** meditation experience

- "Close your eyes" must be the last instruction before audio (no announcement)
- Waves should look like gentle ocean waves, not abstract blobs

- Reflections count should be clear (each text/voice entry = 1 reflection)
- **No EAS Build** — always local Xcode builds

## 7. Code Quality Tooling — Research Assignment for Next Agent

### Context
The user is frustrated with repeated back-and-forth debugging cycles, especially for visual/animation issues. He wants an "inspection agent" that runs after every code change to catch problems before they reach him.

### What the user is NOT looking for:
- **Sentry** — This is runtime error monitoring (production crashes). Not helpful at dev time.
- Simple linting — ESLint is already in the project

### What the user IS looking for:
A comprehensive code quality gate that prevents broken, suboptimal, or visually inconsistent code from being committed.

### Options to evaluate (the next agent should research these thoroughly and recommend the best combination):

#### Automated Static Analysis
| Tool | What | Worth it? |
|:-----|:-----|:----------|
| **Husky + lint-staged** | Pre-commit hooks that run `tsc` + `eslint` automatically | ✅ Table stakes — takes 5 min to set up |
| **ESLint strict config** | Stricter rules (no-any, exhaustive-deps, import order) | ✅ Good baseline |
| **TypeScript strict mode** | `strict: true` in tsconfig if not already | Check current tsconfig |
| **Knip** | Dead code / unused exports detection | ✅ Good for cleanup |
| **Biome** | Ultra-fast linter + formatter (replaces ESLint + Prettier) | Evaluate speed vs ESLint compatibility |

#### Visual / UI Testing
| Tool | What | Worth it? |
|:-----|:-----|:----------|
| **React Native Testing Library** | Component unit tests — verify render output | ✅ High value for logic |
| **Storybook for React Native** | Preview components in isolation | Medium effort, great for design review |
| **Detox** | E2E testing on real simulator | High effort, very valuable for flows |
| **Maestro** | Simpler E2E alternative to Detox | Easier setup, growing community |
| **Percy / Chromatic** | Visual regression testing (screenshot comparison) | Web-focused, limited RN support |
| **Applitools** | AI-powered visual testing | Expensive, enterprise-grade |

#### AI-Powered Code Review
| Tool | What | Worth it? |
|:-----|:-----|:----------|
| **GitHub Copilot code review** | AI review on PRs | If using GitHub PRs |
| **CodeRabbit** | AI code review bot for PRs | Good reviews, free tier available |
| **Sourcery** | AI refactoring suggestions | Python-focused, limited for TS/RN |
| **Custom pre-commit script** | Run `tsc` + `eslint` + custom checks | ✅ Free, fully customizable |

### My recommendation (for next agent to validate):
1. **Husky + lint-staged** — Immediate, zero-cost, prevents broken commits
2. **Maestro E2E tests** — Write critical-path flows (Tadabbur session, audio playback) as automated tests
3. **Knip** — Find and remove dead code
4. **Stricter ESLint rules** — Tighten the safety net

The next agent should evaluate these options against QuranNotes' specific needs (React Native, Expo, solo developer, iOS-only) and recommend + implement the best combination.
