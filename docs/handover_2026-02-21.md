```typescript
// LIGHT MODE
BrandTokens.light.bgMain          = '#F8F5FF'  // 60% of UI — screen backgrounds
BrandTokens.light.bgSurface       = '#EEE9FA'  // 30% of UI — cards, panels, text boxes
BrandTokens.light.textPrimary     = '#1C1033'  // Main headings/body text  (15.4:1 vs bgMain)

BrandTokens.dark.textSecondary   = '#9E9AC8'
```

The MD3 theme colors (`theme.colors.*`) have been updated to match. So many components using `theme.colors.primary`, `theme.colors.onBackground`, etc. will automatically pick up the new values. The explicit `BrandTokens` export exists for components that need non-theme colors (hardcoded hex in StyleSheets).

## 4. Color Migration — Implementation Plan

### Strategy

The MD3 theme already reflects the new tokens. Migration focus is:

1. **Hardcoded raw hex values in StyleSheets** — replace with BrandTokens
2. **Components using old-primary (#5B7FFF)** — ensure they now use `theme.colors.primary` (which is #6246EA)
3. **Components using old-muted-text (#6B7C93, #94A3B8)** — ensure they use `theme.colors.onSurfaceVariant` (now #5E5A98)

### Usage Pattern for New Components

```typescript
import { BrandTokens } from '../../theme/DesignSystem';
import { useTheme } from 'react-native-paper';

const MyComponent = () => {
    const theme = useTheme();
    const T = theme.dark ? BrandTokens.dark : BrandTokens.light;
    return (
        <View style={{ backgroundColor: T.bgMain }}>
            <Text style={{ color: T.textPrimary }}>Main text</Text>
            <Text style={{ color: T.textSecondary }}>Caption</Text>
        </View>
    );
};
```

## 6. Critical Rules — Must Follow

### DO
- Use `theme.colors.primary` for interactive elements — maps to #6246EA
- Use `theme.colors.onBackground` for main headings — maps to #1C1033
- Use `theme.colors.onSurfaceVariant` for captions — maps to #5E5A98
- Use `theme.colors.surfaceVariant` for lavender panels — maps to #EEE9FA
- Use `BrandTokens.light/dark` for absolute hex when needed outside theme
- Always use **white text** on `accentPrimary` buttons

### DO NOT
- Do not use `#5B7FFF` anywhere — old primary color
- Do not use `#6B7C93` or `#94A3B8` — old blue-grey secondary text
- Do not use `#FAF5FF` or `#F3EEFF` — old background values
- Do not put dark text on `accentPrimary` backgrounds (contrast fails — use white)

### Gradient Exception
These gradient arrays are explicitly allowed (documented in color-system.md):
- Dark widget hero: `['#2D1665', '#1A0940']`
- Verse of Day light: `['#8B5CF6', '#7C3AED', '#6D28D9']`
- Morning Adhkar: amber gradient
- Evening Adhkar: navy — `['#1E3A8A40', '#1E40AF50']` (intentional temporal distinction)
