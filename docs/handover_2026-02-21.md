## 4. Critical Context

### Widget Infrastructure (MUST KNOW)
- **App Group:** `group.com.mahmoudahmedalaa.qurannotes` — must be enabled in Apple Developer portal AND Xcode for both the main app target and the widget extension target
- **Widget data bridge:** `modules/widget-bridge/` — Expo native module (`WidgetBridgeModule.swift`) that writes to `UserDefaults(suiteName: appGroup)` and triggers `WidgetCenter.shared.reloadAllTimelines()`
- **Widget target:** `targets/widget/` — configured via `@bacons/apple-targets` plugin, `expo-target.config.js`
- **Apple Team ID:** `2S42RLH67Y` in `app.json`
- **Prebuild required:** After any Swift/config changes: `npx expo prebuild -p ios --clean`

### Widget Visual Fix (The Key Bug)
The `containerBackground(.clear, for: .widget)` + internal dark `ZStack` gradient caused a black border ring artifact on the home screen. **The fix is using `containerBackground(gradient, for: .widget)` directly** so the gradient IS the system background. iOS 16 fallback uses `ZStack { gradient; content }`. All 3 widgets now use this pattern.

### SwiftUI `TimePalette` — Shared Struct Location
`TimePalette` struct is defined in `DailyVerseWidget.swift` and used by all 3 widget files. This works because all Swift files in a widget target compile together as one module. Do NOT duplicate it.

### Color Extension Location
`Color(hex:)` extension is in `DailyVerseWidget.swift`. PrayerWidget and KhatmaWidget use it from the same module.

### Onboarding Architecture
- **OnboardingContext** (`src/infrastructure/onboarding/OnboardingContext.tsx`) — tracks current step. Step count is implicitly 6 now (context tracks step numbers but no hardcoded max — just update `goToStep()` calls consistently)
- **New flow:** `index` → `listen` → `record` → `widgets` → `reminders` → `premium`
- `pick-surah` still registered in `_layout.tsx` but skipped — can be removed in cleanup

### User Design Preferences (Strong Opinions)
- **No grey/neutral surfaces** — always brand gradients or vivid colors
- **Vivid, modern, premium** — not "minimum viable product" aesthetics
- **Dark mode must look premium** — no grey bleed-through from `theme.colors.surface`
- **Fewer onboarding screens** — strongly prefers concise flows
- **Widgets should look like Apple's own widgets** — rich, vivid, NOT dark and flat

### Data Sync Into Widgets
Three contexts call `WidgetBridge.setWidgetData()` and `.reloadAllWidgets()`:
- `DailyVerseCard.tsx` → syncs verse data
- `PrayerContext.tsx` → syncs next prayer + countdown
- `KhatmaContext.tsx` → syncs completed juz count
