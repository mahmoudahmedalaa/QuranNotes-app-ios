## 4. Critical Context

### Architectural Decisions
- **Auto-advance over celebration card:** User explicitly wanted the trophy card removed. Auto-advance uses a `useEffect` with a 600ms `setTimeout` and light haptic. If the user is gated (paywall), auto-advance does NOT fire — the gated user must unlock first.
- **Stacked card spacing:** The tilted "ear" cards that peek behind the front card stick out at the top, so the `stackContainer.marginTop` needs to be larger than typical padding (currently 40px). This is not a bug.
- **CatchUpBanner messages:** Messages are picked deterministically via `completedCount % pool.length` for stable display (no flicker on re-render).

### User Preferences
- Prefers **purple** (brand color) over gold for UI accents
- Prefers **calming, encouraging** language — no stressful countdowns
- Dislikes large celebration cards — prefers subtle, automatic transitions
- Wants **premium gradients** (left→right fades) over solid color backgrounds
- Values clean spacing and alignment — English/Arabic names should share one row

### Key Constants & Colors
- `ACCENT.green`: `#10B981` (completed state)
- `ACCENT.current`: `BrandTokens.light.accentPrimary` (current juz — purple)
- Completed card gradient: `['#DCFCE7', '#E8FAF0', '#F8FDF9', '#FFFFFF']` left→right
- Progress ring gradient: gold → green (defined in ProgressRing.tsx via SVG LinearGradient)
