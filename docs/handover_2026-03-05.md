## 2. Visual Bugs — Root Cause Analysis

### Bug 1: "Weird purple line" on Favorites page and Category Detail
**Location**: `src/features/hadith/presentation/HadithLibraryScreen.tsx`  
**Root cause**: `detailAccent` style (line ~254, ~697-701) — a 4x36px solid colored bar in the header:
```tsx
<View style={[styles.detailAccent, { backgroundColor: accentColor }]} />
```
```tsx
detailAccent: {
    width: 4,
    height: 36,
    borderRadius: 2,
}
```
**Fix**: Remove this bar entirely, or replace with a subtle gradient header accent.

### Bug 2: "Weird colored vertical line that looks like a scroll bar but doesn't act like it"
**Same element as Bug 1** — the `detailAccent` View sitting in the top-right of the header. It appears as an orphaned colored stripe that looks like a broken scrollbar.

### Bug 3: No actual scroll indicator
**Root cause**: `showsVerticalScrollIndicator={false}` on both FlatList instances (lines ~261, ~435).  
**Fix**: Set `showsVerticalScrollIndicator={true}`.

### Bug 4: Hadith cards blend together — no borders, no separation
**Root cause**: `hadithCard` style (lines ~709-713) has no `borderWidth`, no `borderColor`:
```tsx
hadithCard: {
    borderRadius: BorderRadius.lg,
    padding: Spacing.lg,
    paddingBottom: 0,
}
```
Only `Shadows.sm` is applied (too subtle). No visual boundary between cards.

### Bug 5: Text overflow in Hadith Library topic cards
**Root cause**: `topicName` style allows `numberOfLines={2}` which can overflow for long names like "Brotherhood & Community" or "Forgiveness & Repentance" within the fixed 140px card height.

### Bug 6: "Weird colored squares" in filter modal
**Root cause**: The `modalOptionIcon` (lines 823-828) uses topic color + 14% opacity as background, but the icon size (16px) inside a 32x32 container creates awkward visual weight.

### Bug 7: Yellow/amber text in DailyHadithCard doesn't fit any background
**Location**: `src/features/hadith/presentation/DailyHadithCard.tsx`  
**Root cause**: `BOOKMARK_ACTIVE = '#F59E0B'` (amber yellow) is used for:
- "Explore Hadith Library" link text and icons (line ~249-251)
- Active bookmark heart icon color (line ~184)
- Refresh badge background (line ~306)
- The lock icon when refresh is exhausted (line ~166)

This amber color clashes against ALL the earth-tone gradient backgrounds (`getHadithGradient`). It looks out of place.

**Fix**: Replace amber accent elements with white (`TEXT_PRIMARY` or `TEXT_SECONDARY`) to match the rest of the card's text hierarchy. The "Explore Hadith Library" row should use `rgba(255,255,255,0.15)` background with white text, similar to how "Hadith of the Day" label is styled.

## 3. Brand Identity & Design System Context

### Design Philosophy
The app is inspired by **Calm/Headspace** — serene, minimal, luxurious:
- Soft, calming colors (avoid harsh saturation)
- Layered depth with shadows
- Generous whitespace
- Subtle gradients

### Color System (60-30-10 Rule)
- **60%** — Background: `#F8F5FF` (light lavender) / `#09090B` (dark)
- **30%** — Surface: `#FFFFFF` (pure white) / `#18181B` (zinc 900)
- **10%** — Accent: `#6246EA` (primary purple) / `#A78BFA` (soft violet)

## 6. Share Functionality (Missing from Category Detail)

The `DailyHadithCard.tsx` already implements sharing via `ViewShot + expo-sharing`. The pattern:

```tsx
import ViewShot from 'react-native-view-shot';
import * as Sharing from 'expo-sharing';

<ViewShot ref={viewShotRef} options={{ format: 'png', quality: 1.0 }}>
  {/* Card content */}
</ViewShot>

const handleShare = async () => {
    const capture = viewShotRef.current?.capture;
    const uri = await capture();
    if (await Sharing.isAvailableAsync()) {
        await Sharing.shareAsync(uri, {
            mimeType: 'image/png',
            dialogTitle: 'Share Hadith',
        });
    }
};
```

Add a "Share" action button alongside "Save" and "Use for Today" in the card action bar.
