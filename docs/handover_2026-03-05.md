## 4. Critical Context

### Architectural decisions
- **Local curated data only** — no API calls for hadiths. All 94 hadiths bundled in `CuratedHadiths.ts`
- **Usage-based gating** — free users get full daily hadith experience but hit friction (3 refreshes, 3 bookmarks, locked library) that drives conversion
- **5 distinct paywall entry points** with contextual reasons: `hadith-library`, `hadith-bookmarks`, `hadith-refresh` (plus 2 from the same library route)

### User preferences
- No emoji icons next to section titles
- Premium aesthetics using brand design tokens (`BrandTokens`, `Spacing`, `BorderRadius`, `Shadows`, `Typography`)
- Libraries/packages preferred over custom implementations
- End-to-end thinking required — no lazy/incomplete implementations
- Everything must lead to profitability

### Bugs solved
- Topic cards were misaligned → fixed with `Dimensions.get('window')` for calculated CARD_WIDTH
- Pro banner wasn't tappable → wrapped in Pressable with paywall route
- Calendar icon was confusing → replaced with "Set as Today" text button
- Notification bell had no value → removed entirely + cleaned up stale paywall case
