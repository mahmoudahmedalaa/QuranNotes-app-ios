## 4. Critical Context

### Architectural Decisions
1. **Primitive Float Uniforms over Array Uniforms**: Reanimated's `useDerivedValue` cannot safely serialize JavaScript arrays across the JSI bridge to Skia's C++ runtime. Passing `vec4` arrays results in all-black renders. The solution is to pass each color channel as a separate `float` uniform and reconstruct `vec3` inside the SKSL shader's `main()` function. **DO NOT revert this to array-based uniforms.**

2. **Custom Animation Requires Explicit translateX**: `react-native-reanimated-carousel` completely overrides default layout when using `customAnimation`. If you only specify `scale`/`opacity`/`rotateY` without `translateX`, all items stack at position 0. The current implementation uses:
   ```typescript
   translateX = interpolate(value, [-2,-1,0,1,2], [-W*1.8, -W*1.1, 0, W*1.1, W*1.8], Extrapolation.CLAMP)
   ```

3. **Mood Color Palettes**: Each mood has a `palette` of 4 hex colors in `Mood.ts`. These are NOT gradients — they are the 4 blob center colors for the mesh gradient shader. Example for "Grateful": `['#FFD700', '#FFA500', '#FFFACD', '#FFE4B5']` (warm golds/ambers).

### Tricky Bugs Encountered & Solved
- **Black gradient squares**: Caused by JSI bridge serialization failure with array uniforms → fixed with primitive floats.
- **Carousel items overlapping at center**: Missing `translateX` in `customAnimation` worklet → added explicit X-axis interpolation.
- **`agvtool` warning "Cannot find YES"**: Benign warning from Xcode project having a `GENERATE_INFOPLIST_FILE = YES` entry that `agvtool` tries to resolve as a path. Does not affect build.
