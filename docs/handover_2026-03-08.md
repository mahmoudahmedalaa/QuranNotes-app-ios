## 4. Critical Context

- **NO EAS BUILDS** — User uses `./build-ios.sh` + Xcode Organizer for TestFlight
- Firebase uses **JS SDK** (not `@react-native-firebase/ai`) — imports are `firebase/ai`
- Tafsir data uses **static require()** via generated map — React Native bundler can't handle dynamic requires
- User cares deeply about visual polish — icons must be consistent, animations must be subtle
- The header-bar icons in SurahDetail (pencil, mic, study mode) intentionally use `theme.colors.primary` (purple) because they sit in white contained-tonal buttons — that's a different design context from the verse-level inline icons
